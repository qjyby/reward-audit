## 【魔域】25年冰雪派对 案例集（真实案例）

### 案例编号：CASE-001
- **问题类型**：价值统计缺失
- **发生场景**：在线礼包、每日活跃礼包
- **错误表现**：价值列全部填写为 0，但实际期望价值分别为 40/59.5/70/79 筹码
- **影响**：
  - 读者无法直观了解各档礼包的 MS 价值
  - 后续成本统计数据不准确
  - 汇总表的计算会受影响
- **正确做法**：
  - 填入实际期望值（计算：筹码数 ÷ 汇率，如 40 筹码 ÷ 2.3 = 17.4 MS）
  - 或备注该列由汇总 Sheet 统一计算，此处仅参考
- **关键数字**：40、59.5、70、79 筹码 = 17.4、25.9、30.4、34.3 MS

### 案例编号：CASE-002
- **问题类型**：配置逻辑不一致
- **发生场景**：积分目标礼包
- **错误表现**：
  - A 列（档次列）：225、675、1050、1425、1800
  - C 列（积分门槛）：250、650、1050、1450、1850
  - 两列数值不一致，差异无规律
- **影响**：
  - 研发和运营理解混乱（A 列是什么含义？）
  - 玩家配置表可能与实际不符
- **正确做法**：
  - 明确 A 列的含义（例如是否包含小数位？）
  - 如果 A 列无用，直接删去，仅保留积分门槛列 C
  - 如果 A 列有特殊含义，在表头添加备注
- **案例启示**：配置表中每列都要有明确含义，避免歧义

### 案例编号：CASE-003
- **问题类型**：时效性降级风险
- **发生场景**：积分目标礼包 - 皇家节日徽章
- **错误表现**：时效期后不可选徽章，只能改选恒晶石（1000 个 = 20 MS），价值缩水至 25%
- **影响**：
  - 玩家忘记在时效期内开启，将获得大幅低于预期的价值
  - 对玩家体验造成负面影响（惩罚感强）
  - 游戏经济失控（实际价值产出与配置严重不符）
- **正确做法**：
  - 增加显眼的时效期提醒（在游戏 UI 中高亮显示）
  - 或提升超时替换道具的价值（如 1200 恒晶石 = 24 MS）
  - 或允许玩家在时效期后仍能选择原奖励（但需要额外操作）
- **严重等级**：🔴 中等（直接影响玩家收益预期）

### 案例编号：CASE-004
- **问题类型**：奖励断层
- **发生场景**：日榜奖励 21-50 名
- **错误表现**：
  - 日榜 21-50 名：仅有 300 个筹码，总价值 = 0 MS
  - 日榜 11-20 名：有超星灵药精华，总价值 = 13.5 MS
  - 段位间奖励断层明显
- **影响**：
  - 21-50 名玩家奖励感知低，参与排行榜活动的动力严重不足
  - 容易导致排行榜活跃度骤降
- **正确做法**：
  - 给 21-50 名补充少量超星灵药精华（如 5 个 = 6.75 MS）
  - 奖励曲线应平滑递减，避免陡峭断层
  - 参考：11-20 名 13.5 MS → 21-30 名 8 MS → 31-50 名 4 MS（示例）
- **案例启示**：高难度区间的排名分布应对应平滑的奖励递减，不可有太大落差

### 案例编号：CASE-005
- **问题类型**：成本核算错误
- **发生场景**：总榜第 1 名 - 每档合计
- **错误表现**：
  - 总榜第 1 名每档合计列 = 0
  - 但个人价值 = 1943 MS
  - 结果：汇总表中服务器成本少算了 1943 MS
- **影响**：
  - 汇总成本数据失准，影响后续经济规划
  - 无法准确评估这个活动对游戏经济的冲击
- **正确做法**：
  - 将 R46 每档合计填入 1943
  - 建议用公式：`=SUM(前面每档第1名的个人价值)`
  - 避免手动填写，防止遗漏
- **案例启示**：所有汇总行都应该用公式而非硬编码数字

### 案例编号：CASE-006
- **问题类型**：价值统计缺口
- **发生场景**：筹码商店 - 1 小时经验包/祝福礼包
- **错误表现**：
  - 折扣价 = 0（应该填写实际消耗筹码的价值）
  - 消耗 50 筹码（约 21.7 MS）
  - 价值核算中完全缺失这笔成本
- **影响**：
  - 如果这个商品是免费赠送或额外奖励的一部分，成本统计不全
  - 玩家可能通过这个商品 "套利"
- **正确做法**：
  - 设置象征性折扣价（如 0.5 MS）
  - 或直接从消耗商品列表中移出
  - 或明确这笔成本由其他预算承担
- **案例启示**：每个消耗商品都必须有对应的价值，不能为 0

### 案例编号：CASE-007
- **问题类型**：定价逻辑偏低
- **发生场景**：筹码商店 - 高战力道具
- **错误表现**：
  - 95 级神火碎焰：价值比 = 0.3375（低于标准 0.5）
  - 灵魂晶石小礼包：价值比 = 0.375（低于标准 0.5）
  - 现象：高战力区道具定价普遍偏低
- **影响**：
  - 高战力玩家兑换这些道具时获得的价值缩水
  - 可能导致高端玩家流失（感觉被坑）
  - 影响游戏经济的公平性
- **正确做法**：
  - 说明定价逻辑（为什么这些道具要更便宜？）
  - 或统一调整至 0.45-0.5 区间，与其他区间保持一致
  - 建议附注备注：例如"95 级道具稀有度低，定价相应下调"
- **案例启示**：商品定价应有统一的逻辑体系，不能随意

## 检查清单总结

基于以上 7 个真实案例，审核时重点检查：

- [ ] **价值统计**：所有价值列是否填写完整（不能为 0 或空白）
- [ ] **配置一致性**：相关列的数值逻辑是否一致
- [ ] **时效性风险**：有无时效期限的道具价值是否会在期后暴跌
- [ ] **奖励梯度**：排名/难度段位间的奖励是否平滑递减
- [ ] **汇总行**：所有汇总/合计都是否用公式计算
- [ ] **商品定价**：所有消耗商品的价值比是否在合理范围内
- [ ] **逻辑备注**：配置中是否有不清楚的列或异常值（应添加备注）
