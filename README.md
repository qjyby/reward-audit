# Reward Audit Skill - 游戏奖励审核助手

一个专业的游戏数值策划审核工具，用于检查任务/活动奖励配置的**合理性**和**逻辑正确性**。

---

## 快速开始

### 1. 安装依赖

```bash
pip install openpyxl pandas python-docx pillow
```

### 2. 使用方法

**方法A：用 AI 审核 Excel 文件（推荐）**

1. 告诉 AI：我有个 Excel 奖励文件需要审核
2. 上传你的 `.xlsx` 文件
3. AI 会自动分析每张工作表，找出问题，生成**带截图的 Word 报告**

**方法B：手动运行脚本**

```bash
python scripts/gen_audit_report_v2.py
```

修改脚本里的 `XLSX` 路径为你的文件，运行即可。

## 核心功能

### 1. 全面的错误检查

- ✅ 计算错误（总数 ≠ 各项加总）
- ✅ 逻辑错误（条件与奖励不匹配）
- ✅ 奖励缺失（某个档位没有奖励）
- ✅ 奖励重复（同一目标被多次奖励）
- ✅ 上下限缺失（无上限的资源产出）
- ✅ 奖励倒挂（低难度获得更好奖励）
- ✅ 汇率错误（不同货币换算不合理）

详见：[SKILL.md](./SKILL.md) 第 51-63 行

### 2. 自动生成报告

生成的 Word 报告包含：
- **截图**：每个问题的 Excel 表格数据（高亮显示）
- **描述**：问题是什么、为什么是问题
- **影响**：对游戏经济/玩家的影响
- **建议**：具体的修改方案

示例报告：`奖励审核报告_冰雪派对_带截图版_V2.docx`

### 3. 参考数据管理

- `references/game_economy.md` - 游戏经济体系参考（货币汇率、道具价值等）
- `references/case_library.md` - 典型问题案例库（持续积累）

## 常见问题

### Q: 报告里没有图片怎么办？

**A:** 确保用的是 `gen_audit_report_v2.py`（改进版），而不是 `audit_to_word.py`（原始版）。

改进版用 PIL 生成图片，更快更稳定。

### Q: 脚本报错 "too many requests"？

**A:** 这是 API 限制问题。等 1-2 分钟后再试。

或者分步骤运行：
1. 先生成截图
2. 确认成功后再生成报告

### Q: 怎么自定义检查项？

**A:** 修改 `gen_audit_report_v2.py` 里的 `issues` 列表，改成你要检查的工作表和范围即可。

### Q: 生成的报告在哪里？

**A:** 
- Word 报告：`E:\桌面\奖励审核报告_*.docx`
- 截图图片：`C:\Users\Administrator\Desktop\screenshots\issue_*.png`

## 进阶使用

### 1. 添加游戏经济参考数据

编辑 `references/game_economy.md`，填入你游戏的：
- 货币名称和汇率（如：1000 筹码 = 20 MS）
- 道具价值参考（如：超星灵药精华 = 1.35 MS）
- 各阶段资源产出预期

这样审核时能更精准地判断奖励是否合理。

### 2. 积累典型案例

发现新的错误模式后，写入 `references/case_library.md`：

```markdown
### 案例编号：CASE-002
- **问题类型**：奖励倒挂
- **发生场景**：日榜 21-50 名奖励断层
- **错误表现**：仅有 300 个筹码，与前 20 名（13.5 MS）断层明显
- **影响**：该段位玩家奖励感知低，参与动力不足
- **正确做法**：补充少量超星灵药精华，弥合断层
```

## 技术细节

### 截图生成方式

**原始方案**（audit_to_word.py）：
- 使用 matplotlib 渲染完整表格
- 优点：图形化更好看
- 缺点：耗时长、容易卡住

**改进方案**（gen_audit_report_v2.py）- 推荐使用：
- 使用 PIL 直接生成表格
- 优点：快 100 倍、稳定可靠
- 缺点：图片样式比较简洁

### 编码兼容性

- 避免 emoji（✓ ✗ 会导致 GBK 编码错误）
- 改用纯文字标记（[OK] [ERROR]）
- 脚本文件头：`# -*- coding: utf-8 -*-`

### 文件验证

生成后自动检查：
```python
from docx import Document
doc = Document(report_path)
image_count = sum(1 for rel in doc.part.rels.values() if 'image' in rel.reltype)
print(f'成功：{image_count} 张图片已嵌入')
```
